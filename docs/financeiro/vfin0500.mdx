# VFIN0500 — Relatórios Financeiros

> Central de relatórios R01–R19: livros fiscais, DRE, aging, curva ABC e produtos.

**Código fonte:** `src/components/screens/financeiro/Vfin0500Page.tsx`

## Objetivo

Consolidar 19 relatórios financeiros e fiscais em uma única interface unificada. A tela utiliza um sistema inteligente de parâmetros dinâmicos: cada relatório declara seu tipo (range, none, entity, item) que determina quais controles de filtro são exibidos. O resultado pode ser exibido em formato "single" (tabela chave-valor) ou tabular (colunas dinâmicas).

## Pre-requisitos

- Dados financeiros e fiscais registrados no sistema conforme o relatório desejado.
- Para relatórios com período: ter movimentações no intervalo informado.
- Para relatórios por entidade: ter o ID do fornecedor ou cliente.
- Para R05 (DRE): plano de contas (VFIN0120) configurado.

## Passo a passo

1. Acesse **Financeiro > Relatórios** (VFIN0500).
2. Selecione o **Relatório** desejado na lista suspensa (R01 a R19).
3. Os campos de filtro aparecem dinamicamente conforme o tipo de relatório:
   - **Período:** informe Data Início e Data Fim.
   - **Entidade:** informe o ID do fornecedor ou cliente.
   - **Item:** informe o código do item.
   - **Sem filtro:** nenhum parâmetro adicional necessário.
4. Clique em **Gerar**.
5. O resultado e exibido em tabela. Para relatórios "single", o formato e chave-valor. Para tabulares, colunas dinâmicas.
6. Use **Exportar** para gerar planilha do resultado.

## Campos

| Campo | Tipo | Obrigatório | Opções | Função |
|-------|------|-------------|--------|--------|
| Relatório | seleção | Sim | R01 a R19 | Seleção do relatório |
| Período Início | data | Condicional | — | Para relatórios com período |
| Período Fim | data | Condicional | — | Para relatórios com período |
| ID Fornecedor / Cliente / Item | texto | Condicional | — | Para relatórios por entidade/item |

## Observações importantes

- O tipo de filtro e gerenciado dinamicamente pelo sistema; campos irrelevantes para o relatório selecionado são ocultados automaticamente.
- O R05 (DRE) depende da estrutura do plano de contas (VFIN0120) para consolidação correta.
- Relatórios de aging (R09, R10) não necessitam de filtro de período — exibem a situação atual dos títulos.
- O formato de saída (single vs tabular) e definido pelo tipo de dados que o relatório retorna.
