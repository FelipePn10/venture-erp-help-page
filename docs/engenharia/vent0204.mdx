# VENT0204 — Cadastro de Grupo PDM

> Primeiro nivel da hierarquia de configuracao de itens PDM. Define os agrupamentos principais que organizam os itens configuraveis e suas variacoes.

## Objetivo

Estabelecer o primeiro nivel da hierarquia de configuracao de produtos no modelo PDM (Product Data Management). Os Grupos PDM funcionam como categorias principais que agrupam itens configuraveis com caracteristicas similares, servindo como raiz da arvore de configuracao. A partir do grupo, derivam-se os modificadores e atributos que definem as possiveis variacoes do produto. Esta e a tela fundamental para iniciar a parametrizacao de qualquer produto configuravel no sistema.

## Pre-requisitos

- [VENT0200 (Cadastro de Itens)](/engenharia/vent0200) — Itens que serao associados aos grupos PDM devem estar cadastrados.
- [VITE0115 (Modificadores PDM)](/engenharia/vite0115) — Modificadores que serao vinculados aos grupos PDM ja devem estar definidos.

## Passo a passo

1. Acesse a tela VENT0204 pelo menu Engenharia > Cadastro de Grupo PDM.
2. Clique em **Novo** para iniciar o cadastro de um novo grupo.
3. Preencha o **Codigo do Grupo** com um identificador unico e significativo.
4. Informe a **Descricao** do grupo, detalhando o tipo de produto que ele abrange (ex: "Cadeiras", "Mesas", "Sofas").
5. Selecione o **Item Base** que servira como modelo para os itens gerados a partir deste grupo.
6. Defina a **Mascara** de codificacao que determinara como os codigos dos itens configurados serao gerados.
7. Marque os modificadores PDM que estarao disponiveis para este grupo na tabela de **Modificadores Vinculados**.
8. Defina a **Ordem** de exibicao dos modificadores, que determinara a sequencia de escolha durante a configuracao.
9. Clique em **Salvar** para persistir o grupo PDM e suas associacoes.

## Campos

| Campo | Tipo | Obrigatorio | Opcoes | Descricao |
|-------|------|-------------|--------|-----------|
| Codigo do Grupo | texto | Sim | — | Identificador unico do grupo PDM |
| Descricao | texto | Sim | — | Nome descritivo do grupo PDM |
| Item Base | select | Nao | Lista de itens | Item modelo para itens gerados pelo grupo |
| Mascara | texto | Nao | — | Padrao de codificacao para itens configurados |
| Situacao | select | Sim | Ativo / Inativo | Status do grupo PDM no sistema |
| Modificadores Vinculados | tabela | Nao | Lista de modificadores | Modificadores PDM disponiveis para este grupo |
| Ordem Modificador | numero | Nao | — | Ordem sequencial do modificador na configuracao |
| Obrigatoriedade | toggle | Nao | — | Define se o modificador e obrigatorio na configuracao |

## Observacoes importantes

- O **Grupo PDM** e o ponto de partida da hierarquia de configuracao. Sem ele, nao e possivel vincular modificadores e atributos aos itens.
- A **Mascara** de codificacao define o padrao automatico de geracao de codigos para itens configurados, garantindo unicidade e rastreabilidade.
- O **Item Base** e utilizado como referencia para herdar caracteristicas comuns a todos os itens gerados a partir do grupo, como unidade de medida e tipo de planejamento.
- A ordem dos modificadores influencia a experiencia do usuario no configurador, determinando em qual sequencia as escolhas serao apresentadas.
- Grupos inativos nao ficam disponiveis para novas associacoes, mas os itens ja gerados a partir deles continuam funcionando normalmente.
- E recomendavel criar grupos PDM com nomes claros e objetivos, facilitando a localizacao por outros usuarios durante a configuracao de produtos.
