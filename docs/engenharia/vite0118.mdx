# VITE0118 — Regras Itens Configurados

> Motor de regras condicionais que define restricoes de compatibilidade, dependencia e exclusao entre atributos de itens configurados.

## Objetivo

Permitir a criacao e gestao de regras condicionais que controlam o comportamento do configurador de produtos. Estas regras definem relacoes de dependencia (se escolher A, entao B deve ser escolhido), exclusao (se escolher A, entao B nao pode ser escolhido) e recomendacao (se escolher A, sugira B) entre atributos de diferentes modificadores. As regras garantem que apenas combinacoes viaveis do ponto de vista tecnico e comercial sejam permitidas, evitando erros de configuracao que resultariam em produtos impossiveis de fabricar ou comercializar.

## Pre-requisitos

- [VITE0115 (Modificadores PDM)](/engenharia/vite0115) — Modificadores PDM com atributos cadastrados.
- [VITE0116 (Atributos PDM)](/engenharia/vite0116) — Atributos PDM que serao utilizados nas condicoes das regras.
- [VENT0204 (Cadastro de Grupo PDM)](/engenharia/vent0204) — Grupos PDM para os quais as regras serao aplicaveis.
- [VENT0210 (Estrutura de Produtos)](/engenharia/vent0210) — Estrutura BOM impactada pelas regras de configuracao.

## Passo a passo

1. Acesse a tela VITE0118 pelo menu Engenharia > Regras Itens Configurados.
2. Selecione o **Grupo PDM** para o qual a regra sera aplicada.
3. Clique em **Nova Regra** para iniciar a definicao de uma nova regra condicional.
4. Escolha o **Tipo de Regra**: Dependencia (IF-THEN), Exclusao (NOT WITH), Recomendacao (SUGGEST) ou Restricao Condicional.
5. Defina a **Condicao** (IF): selecione o modificador e o atributo que disparam a regra.
6. Defina a **Acao** (THEN): selecione o modificador e o atributo que serao afetados pela regra.
7. Configure a **Mensagem** que sera exibida ao usuario quando a regra for acionada no configurador.
8. Defina a **Prioridade** da regra para resolver conflitos quando multiplas regras se aplicam simultaneamente.
9. Marque o toggle **Regra Ativa** para que a regra entre em vigor no configurador.
10. Clique em **Salvar** para armazenar a regra.

## Campos

| Campo | Tipo | Obrigatorio | Opcoes | Descricao |
|-------|------|-------------|--------|-----------|
| Grupo PDM | select | Sim | Lista de grupos PDM | Grupo PDM de abrangencia da regra |
| Tipo de Regra | select | Sim | Dependencia / Exclusao / Recomendacao / Restricao | Categoria da regra condicional |
| Condicao — Modificador | select | Sim | Lista de modificadores | Modificador que dispara a condicao |
| Condicao — Atributo | select | Sim | Lista de atributos | Atributo que dispara a condicao |
| Acao — Modificador | select | Sim | Lista de modificadores | Modificador afetado pela regra |
| Acao — Atributo | select | Sim | Lista de atributos | Atributo afetado pela regra |
| Mensagem | texto | Sim | — | Texto explicativo exibido ao usuario no configurador |
| Prioridade | numero | Nao | — | Ordem de avaliacao da regra (menor = maior prioridade) |
| Regra Ativa | toggle | Sim | — | Status de vigencia da regra |
| Severidade | select | Nao | Erro / Aviso / Informativo | Nivel de severidade da mensagem exibida |

## Observacoes importantes

- Regras do tipo **Dependencia** (IF-THEN) forcam automaticamente a selecao do atributo dependente quando a condicao e satisfeita, sem permitir que o usuario desmarque.
- Regras do tipo **Exclusao** (NOT WITH) impedem que dois atributos incompatveis sejam selecionados simultaneamente. Se o usuario tentar, o sistema exibe a mensagem de erro e bloqueia a selecao.
- Regras do tipo **Recomendacao** (SUGGEST) apenas sugerem ao usuario uma combinacao recomendada, mas permitem que ele escolha outra opcao se desejar.
- A **Severidade "Erro"** bloqueia o usuario de prosseguir com a configuracao. **"Aviso"** alerta, mas permite continuar. **"Informativo"** apenas mostra uma dica.
- A **Prioridade** e importante quando duas regras entram em conflito. A regra com menor numero de prioridade prevalece.
- Recomenda-se testar exaustivamente as regras no configurador apos cada alteracao, simulando diferentes combinacoes de atributos para garantir que as restricoes funcionem conforme esperado.
