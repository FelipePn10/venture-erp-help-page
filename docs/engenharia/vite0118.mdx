# VITE0118 — Regras Itens Configurados

> Motor de regras condicionais que define restrições de compatibilidade, dependência e exclusão entre atributos de itens configurados.

## Objetivo

Permitir a criação e gestão de regras condicionais que controlam o comportamento do configurador de produtos. Estas regras definem relações de dependência (se escolher A, entao B deve ser escolhido), exclusão (se escolher A, entao B não pode ser escolhido) e recomendação (se escolher A, sugira B) entre atributos de diferentes modificadores. As regras garantem que apenas combinações viáveis do ponto de vista técnico e comercial sejam permitidas, evitando erros de configuração que resultariam em produtos impossíveis de fabricar ou comercializar.

## Pre-requisitos

- [VITE0115 (Modificadores PDM)](/engenharia/vite0115) — Modificadores PDM com atributos cadastrados.
- [VITE0116 (Atributos PDM)](/engenharia/vite0116) — Atributos PDM que serão utilizados nas condições das regras.
- [VENT0204 (Cadastro de Grupo PDM)](/engenharia/vent0204) — Grupos PDM para os quais as regras serão aplicáveis.
- [VENT0210 (Estrutura de Produtos)](/engenharia/vent0210) — Estrutura BOM impactada pelas regras de configuração.

## Passo a passo

1. Acesse a tela VITE0118 pelo menu Engenharia > Regras Itens Configurados.
2. Selecione o **Grupo PDM** para o qual a regra sera aplicada.
3. Clique em **Nova Regra** para iniciar a definição de uma nova regra condicional.
4. Escolha o **Tipo de Regra**: Dependência (IF-THEN), Exclusão (NOT WITH), Recomendação (SUGGEST) ou Restrição Condicional.
5. Defina a **Condição** (IF): selecione o modificador e o atributo que disparam a regra.
6. Defina a **Ação** (THEN): selecione o modificador e o atributo que serão afetados pela regra.
7. Configure a **Mensagem** que sera exibida ao usuário quando a regra for acionada no configurador.
8. Defina a **Prioridade** da regra para resolver conflitos quando múltiplas regras se aplicam simultaneamente.
9. Marque o toggle **Regra Ativa** para que a regra entre em vigor no configurador.
10. Clique em **Salvar** para armazenar a regra.

## Campos

| Campo | Tipo | Obrigatório | Opções | Descrição |
|-------|------|-------------|--------|-----------|
| Grupo PDM | select | Sim | Lista de grupos PDM | Grupo PDM de abrangência da regra |
| Tipo de Regra | select | Sim | Dependência / Exclusão / Recomendação / Restrição | Categoria da regra condicional |
| Condição — Modificador | select | Sim | Lista de modificadores | Modificador que dispara a condição |
| Condição — Atributo | select | Sim | Lista de atributos | Atributo que dispara a condição |
| Ação — Modificador | select | Sim | Lista de modificadores | Modificador afetado pela regra |
| Ação — Atributo | select | Sim | Lista de atributos | Atributo afetado pela regra |
| Mensagem | texto | Sim | — | Texto explicativo exibido ao usuário no configurador |
| Prioridade | número | Não | — | Ordem de avaliação da regra (menor = maior prioridade) |
| Regra Ativa | toggle | Sim | — | Status de vigência da regra |
| Severidade | select | Não | Erro / Aviso / Informativo | Nível de severidade da mensagem exibida |

## Observações importantes

- Regras do tipo **Dependência** (IF-THEN) forcam automaticamente a seleção do atributo dependente quando a condição e satisfeita, sem permitir que o usuário desmarque.
- Regras do tipo **Exclusão** (NOT WITH) impedem que dois atributos incompatveis sejam selecionados simultaneamente. Se o usuário tentar, o sistema exibe a mensagem de erro e bloqueia a seleção.
- Regras do tipo **Recomendação** (SUGGEST) apenas sugerem ao usuário uma combinação recomendada, mas permitem que ele escolha outra opção se desejar.
- A **Severidade "Erro"** bloqueia o usuário de prosseguir com a configuração. **"Aviso"** alerta, mas permite continuar. **"Informativo"** apenas mostra uma dica.
- A **Prioridade** e importante quando duas regras entram em conflito. A regra com menor número de prioridade prevalece.
- Recomenda-se testar exaustivamente as regras no configurador após cada alteração, simulando diferentes combinações de atributos para garantir que as restrições funcionem conforme esperado.
